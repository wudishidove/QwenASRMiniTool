# v5 訓練結果（pkm-ft-1.7b-v5，2026-07-02）

> 一句話：以「使用者全片手校 diff」為唯一錯誤地圖，golden 含詞片段回訓補零曝光專名（裸催 0→55/53），
> 搭配 app 切分修復，兩輪迭代端到端驗收 PASS。診斷明細見 [`v4全片診斷_v5執行.md`](v4全片診斷_v5執行.md)。

---

## 一、訓練思維（為什麼這樣做）

1. **錯誤地圖來自手校 diff，不是猜測。** `pokemon_srt` repo：HEAD=V4 輸出、working tree=使用者逐句手校
   → `git diff` 就是 84.5 分鐘全片的 ground-truth 錯誤清單（113 處），每一處都能歸因。
2. **先分「模型能修 vs app 才能修」。** 斷句/數字拆行/詞中切斷是 app 層（重訓救不到，v5plan 已證）；
   專名錯誤才進訓練管線。兩軌正交、各自驗收。
3. **禁止錯字替換後處理（使用者鐵則）。** 零曝光詞的錯法不穩定（裸催→裸吹/螺吹/裸槌/裸摔），
   替換表不可能窮舉 → 唯一正解是讓模型「聽得懂」：給訓練訊號。
4. **golden 即資料（使用者拍板）。** 「裸催」全語料曝光=0、golden 出現 53 次 → golden 是唯一來源。
   只切**含目標詞的視窗**（非全片），全片其餘 ~66% 視窗保持乾淨，端到端驗收仍有效。
5. **量化一切、閘門先行。** 每個詞的 train 曝光數先 grep 自證（`make_v5_split` assert）；
   replay 有硬護欄（≤train 10%、per-term R 分級）；驗收前先把 V4 基線跑出來（`compare_golden.py`），
   改善才可證偽。

## 二、優化決策與理由

| 決策 | 理由 |
|---|---|
| 訓練基底回到混合語料（train.clean 系）而非 v4 的純 yt1 | 手校 diff 出現 8 處「域幻覺」（晴天→奇麒麟、傷害→賽亞人）＝v4 過擬合副作用；混合語料是反訊號 |
| replay 分級 R（裸催 R2、零曝光稀詞 R8、v4 三詞 R5-7）而非一律 ×8 | 裸催本身 53 次已密；稀詞才需要高 R；總量壓在 10% 護欄內防傾斜 |
| 已達標詞（風速狗/破滅之光/催眠術）退出 replay | 騰額度給未達標詞，避免無效過取樣 |
| 仆斬將軍只用 golden 影片視窗 anchor（VID_ONLY） | train 曝光已 124，缺的是目標講者語境，全量 replay 會爆護欄 |
| test.v5 依 video_id 重抓（=v4 同三部影片） | 03 重跑後 clip id 全變，沿用舊 manifest 會指錯音檔；同題目才能跨版比較 |
| checkpoint 用 best-by-val（v5r1/v5r2 皆 best 勝 last） | val.v5 為混合集，best-by-val 泛化較佳（v4 時代 val 無難詞只能採 last 的前提已不存在） |
| **golden 片段 lang 一律 zh（v5.1 關鍵修正）** | 生產推理永遠用 Chinese token；v5r1 依 has_japanese_kana 標 ja → 訓練/推理 token 不一致，片假名雜訊（フレンじゃんけん）漏進中文輸出、仆斬將軍 19/24 回退。改 zh 後雜訊 0、仆斬將軍 23/24 → 假設證實 |
| 端到端閘 > 離線閘 | v5.1 在 test.v5 略遜 v5r1（CER 0.1065 vs 0.1017），但 golden 端到端全面勝出 → 以使用者實際使用情境（Fiske 影片）拍板採 v5.1 |

## 三、訓練過程

1. **資料**（`02→03→04`）：新增 `8xmhtLLnvF4`（術語解析 15min）＋「哪些寶可夢應該優先抽」（81min，創作者字幕）
   → train 16.93h/5747 clips，QA gate 剔 1 筆（>30s）。
2. **golden 片段**（新 `make_golden_terms.py`）：手校 SRT+mp3、與 03 相同 merge_cues 視窗、只取含詞窗
   → v5r1 132 clips/22.7min；v5.1 加仆斬將軍＋六位選手名 → **153 clips/26.4min**（佔全片視窗 33.7%）。
3. **split/replay**（新 `make_v5_split.py`/`make_v5_replay.py`）：train.v5=5795 clips/17.0h（含 h51 沿 v4）、
   test.v5 凍結；replay 573 列（9.9%），全 assert 通過。
4. **訓練**：配方凍結（r16/α32/dropout0.05、lr2e-4 cosine、eff batch 32、3ep、bf16/sdpa、不帶 --minutes）。
   v5r1 best_val 0.2844；v5.1（`adapters/p3-1.7b-v5r2`）best_val **0.2805**，約 20 分鐘/輪（RTX 5090）。
5. **app 修復**（`app-gpu.py`，與模型並行）：
   - 斷行器 `.`/`,` 兩側皆數字不切行（修「或 1.5次」→「或 1」+「5 次」）；
   - `_split_long_vad_range` 優先切「連續 ≥3 chunk 低機率」持續靜音（修 醒來/人名 被切在詞中→兩半誤判）。
   - 單元測試（斷行 5 案例＋切點合成訊號）全過。
6. **驗收（opus subagent ×2 輪）**：`transcribe_golden.py`（走 app 完整 process_file 路徑、OpenCC 關、context 空）
   → `compare_golden.py` 對手校 golden 逐詞比對＋CER＋切分健檢。
