## 一、錯誤類別統計（依修正次數排序）

### C1. 零曝光專名（訓練語料 = 0 筆）— **~50 處，佔一半**
| 正確詞 | 誤判為 | 次數 | train.v4 曝光 | golden 出現 |
|---|---|---|---|---|
| **裸催** | 裸吹×38、螺吹×3、裸槌×2、裸摔×2、裸存我、我吹×2 | **~48** | **0** | **53** |
| 堅硬腦袋 | 建議腦袋×2、金勾臂 | 3 | 0 | 3 |
| 急凍鳥 | 極動鳥 | 3 | 0 | 7 |
| 立大功 | （我吹）立大功 | 1 | 0 | 1 |

→ 模型從未見過這些詞，**每次都錯、錯法不定**（裸吹/螺吹/裸槌/裸摔）——這正是「不能用錯字替換後處理」的原因。
**唯一修法＝給訓練訊號**：從 golden sample（手校 SRT+mp3）切片回訓（使用者已拍板）。

→ 訊號弱：**replay 過取樣 + golden 片段補強**。

### C3. 中高曝光專名偶發誤判 — ~8 處（各 1 次）
濁流(38)→鼠王、雪妖女(40)→需要你、蚊香蛙皇(56)→蚊香貓皇、暗影球(76)→暗球、
仆斬將軍(109)→噴火將軍/仆斬雙刃、幽尾玄魚(184)→右旋、風妖精(232)→魔法晶。
→ 多發生在**極短 cue（<1s）或語速快**處＝聲學/切窗問題，非資料缺口。過取樣無效（v5plan 皮可西教訓）；
靠 C6 app 切窗改善 + 新資料的聲學多樣性。**不設逐詞驗收**。

### C4. 日文人名（片假名）— ~12 處
| 正確 | 誤判為 |
|---|---|
| オオニシ ヒロシ | 岡尼西 ヒロシ／Onishi 和 Hiroshi／Only Series／奧尼西+希路西（跨段拆裂）／ウニシ ヘイロシ／奧尼齊 |

→ 模型會輸出片假名（基座多語），但沒看過這些名字。選手名跨賽事重複出現 → golden 片段回訓有長期價值；
但**對全新人名本質上不可解**，列「盡力改善、不保證」。

### C5. 語境幻覺（把一般話聽成寶可夢實體）— ~8 處
由於風速狗→「幽尾玄魚的風之果」、就算有晴天→「奇麒麟」、前面→「奇麒麟」、下回合→「下古河鱷」、
還是一樣→「還是葉羅比」、傷害→「賽亞人」、我了→「本我了」、這邊上→「月光下」。
→ **v4 純 yt1+過取樣的域過擬合副作用**：不確定音段被拉向高頻實體。
修法：v5 訓練基底回到**混合語料（train.clean 系）**、replay 比例設護欄（≤10%、R≤8）。

### C6. 斷句/切分（app 層，模型救不了）— ~8 處
1. **小數點被斷行器吃掉**：「或 1.5 次」→「或 1」＋「5 次」。
   根因＝`app-gpu.py` `_split_to_lines`/`_ts_to_subtitle_lines` 把 `.` 列入 `_EN_SENT_END` → 切行且隱藏。
   **修法：`.`/`,` 兩側都是數字時不當斷點**（保護 1.5、1,000）。
2. **VAD 長段切分切在詞中**：幽尾玄魚醒｜來、事情、中不中、人名（奧尼西｜希路西）跨段拆裂 →
   兩半各自誤判（「又會旋醒」＋「來的機率」）。
   根因＝`_split_long_vad_range` 在 >18s 連續語音用「單一 chunk（32ms）最低機率點」切 → 未必是真靜音。
   **修法：優先找「連續 N chunk 低機率」的持續靜音區間，取其中點；找不到才退回現行單點**。

### C7. 常用詞/同音詞 — ~15 處（不設目標）
畢竟→並、較厚→交厚、誅心→朱星、繼續→基於、唱到→唱導、賽→晒、殘血→閃血、吹個→吃個、沒有中→沒有這種…
→ 泛域聲學誤差，逐詞追打＝打地鼠。期待混合語料+聲學多樣性自然改善；**不做文字替換後處理**（鐵則）。

## 二、v5 修正方案（本輪執行）

**原則**：不做任何輸出端錯字替換；模型端給真訊號、app 端修切窗。配方凍結（r16/alpha32/lr2e-4/3ep/eff batch 32）。

### Track A：資料 + 模型（pkm-ft-1.7b-v5）
1. **新來源入庫**：`8xmhtLLnvF4`（隊伍類型術語，已下載）＋「哪些寶可夢應該優先抽」（mp3+創作者 srt，全圖鑑強度排名 → 大量專名）→ 重跑 02→03→04。
2. **golden 專名片段**（新腳本 `make_golden_terms.py`）：手校 SRT+mp3，只切**含目標詞的 cue 窗**（裸催53/堅硬腦袋/急凍鳥/立大功/極光幕/催眠術/妖精氣場/自我再生/破滅之光/一挑二/三位選手片假名），8–12s 視窗、cue 邊界補「，」與 03 一致 → `golden.terms.jsonl`。
   ⚠️ 驗收影片=訓練來源之一（使用者拍板接受）：只取含詞窗（≈全片 10–15%），端到端驗收仍能檢驗其餘 85% 與斷句。
3. **v5 split**（`make_v5_split.py`）：base=新 train/val（含新兩部＋general 混合），h51 併入 train（沿 v4），test.v5=依 video_id 重抓 {Jwy3M2W987c, VM6LcDCdV1M, NTt8wMs6OUE}（凍結題目、跨版可比）；audio-disjoint assert。
4. **replay.v5**（`make_v5_replay.py`）：terms 檔驅動、依 audio 去重、per-term R≤8、replay 總量 ≤ train 10%。
5. 訓練（不帶 --minutes）→ checkpoint 掃描（epoch0/1/2/last 對 test.v5 CER + golden-terms 逐詞）→ merge+sanitize → `GPUModel/pkm-ft-1.7b-v5`。

### Track B：app 音訊處理（本輪兩點，見 C6）
- `.`/`,` 數字間不斷行（兩處斷行器同步修）。
- `_split_long_vad_range` 改「持續低機率區間優先」。

### 驗收（唯一 GO）
opus subagent 用 v5 + 修好的 app 重轉「裸催王者」mp3 → 與手校 golden 逐 cue 比對：
- 裸催 occurrence recall ≥ 90%（53 處）；堅硬腦袋/急凍鳥/極光幕不再錯；
- C5 幻覺不新增；斷句無詞中切斷、1.5 不再拆；
- test.v5 CER 不比 v4 退步、FLEURS 遺忘 Δ≤+0.03。
不達標 → 調 replay R/補片段 → 重訓迭代。
